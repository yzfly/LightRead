# LightRead 轻阅 · 产品设计

## 定位

给爱读书的人做的**本地优先**阅读器：全格式阅读 + 藏书管理 + OPDS 开放书源。
数据全部在用户设备上——没有账号、没有云端、没有追踪。

## 已确定的关键决策

| 决策点 | 结论 | 理由 |
|--------|------|------|
| 端范围 | M1 桌面 (Tauri 2) + 浏览器 PWA；移动原生放 M3 | Tauri 移动端偏新，先把桌面体验做扎实；手机过渡期用 PWA |
| 壳 | Tauri 2 (Rust) | 产物 ~10MB；Rust 侧原生 HTTP 绕开 OPDS 跨域；文件系统能力 |
| 存储 | 桌面：书籍文件落盘 + SQLite 索引；Web：IndexedDB；上层统一 `LibraryStorage` 抽象 | 桌面用户的书永远看得见、拿得走；两端备份文件互通 |
| 阅读引擎 | foliate-js (MIT) | 一个内核覆盖 EPUB/MOBI/AZW3/FB2/CBZ，自带 CFI、搜索、标注 overlay、OPDS 解析 |
| TXT/MD/HTML | 导入时动态转内存 EPUB | 复用同一套排版引擎；中文章节正则切目录；GB18030 编码回退 |
| zlib 等站点 | 不做直连下载集成 | 版权边界。替代：全格式完美导入 + 通用 OPDS 书源框架，用户可自行添加任何源 |
| KFX | 明确不支持 | 重 DRM，法律与技术上均不可行，文档引导用 Calibre |

## 信息架构

```
藏书 (书架网格 / 导入 / 搜索排序 / 进度)
 └─ 阅读器
     ├─ foliate: EPUB MOBI AZW3 FB2 CBZ + 转换后的 TXT/HTML/MD
     │   目录 / 进度条 / 排版设置 / 三主题 / 高亮标注 / 书内搜索 / 键盘翻页
     └─ PDF: 连续滚动 + 懒渲染 / 缩放适宽 / 页码记忆
书源 (OPDS)
 └─ 内置: 古登堡 / Standard Ebooks; 自定义: 任意 OPDS URL (Calibre 等)
     目录浏览 / OpenSearch 站内搜索 / 分页加载 / 一键下载入库
设置
 └─ 存储后端信息 / zip 备份恢复 / Web 跨域代理 / 阅读偏好重置 / 关于
```

## 视觉

字节跳动风亮色系：`#1664FF` 主色、`#F7F8FA` 底、白卡片、8/12px 圆角、克制阴影。
阅读区独立三主题（亮/纸/夜），不影响应用外壳的亮色调性。

## 工程

- Vue 3 + TypeScript + Vite + Pinia；hash 路由（兼容 Tauri file 协议）
- PWA (vite-plugin-pwa)，构建即产出可安装的离线应用
- E2E 冒烟：`npm run build && npm run preview & npm run e2e`（Playwright，10 步覆盖导入→阅读→搜索→持久化）
- 桌面构建需 Rust 工具链：`npm run tauri dev / build`

## 路线图

- **M1 (已完成)**：全格式阅读、藏书、OPDS + arXiv 书源、书源鉴权、网络代理（HTTP/HTTPS/SOCKS4/5）、备份、PWA、Tauri 壳、自动阅读、PDF 双页
- **M2 (已完成)**：TTS 听书（中文优先）、笔记想法、书签、阅读时长统计
- **M2.5**：批量管理、移动端布局打磨、Calibre 书库目录直读、Edge 神经网络音色（可选在线引擎）
- **M3**：iOS/Android 构建、DjVu / CBR / CHM、WebDAV 同步
